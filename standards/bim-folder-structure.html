BIM Project Folder Structure

A standardized folder structure ensures every team member can locate files quickly, maintains consistency across projects, and streamlines collaboration between disciplines.

Purpose

A standardized folder structure ensures every team member can locate files quickly, maintains consistency across projects, and streamlines collaboration between disciplines.

Hierarchy Overview

1 Project Root

Named with the project number. Contains all discipline and document folders.

PROJECT NUMBER

2 Discipline Folders

Organized by design phase: GN, BIM, CD, PD, DD, TD, IFC

02-ARC, 02-BIM, 03-STR...

3 BIM Subfolder

Contains working models, linked files, exports, and coordination documents. Structured for efficient model management.

02-BIM / 01-MODELS (WIP)
Example Path
PROJECT NUMBER / 02-ARC / 02-BIM / 01-MODELS (WIP)

Discipline Folders

The project root contains numbered folders for each discipline and document type. This structure allows for clear organization by project phase.

Archived files and superseded documents

BIM Subfolders (02-BIM)

The BIM folder within each discipline contains working models, linked files, exports, and coordination documents. This structure supports efficient model management and collaboration.

01-GN
General Information
02-BIM
BIM Models & Data
03-CD
Concept Design
04-PD
Preliminary Design
05-DD
Detailed Design
06-TD (OPT)
Tender Design (Optional)
07-IFC
Issued for Construction
